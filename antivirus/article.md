# Антивирус в песочнице

Скоро сказка сказывается, да не скоро дело делается. Быстро Linux устанавливается, да настраивается
медленно. Особенно в случае чего-нибудь эдакого... Взять, к примеру,
[Qubes OS](https://www.qubes-os.org/). Это не очередной дистрибутив, а целая операционная система со
своими особенностями, достоинствами и недостатками. Но для антивируса в песочнице она подходит очень
хорошо. Дело в том, что в мире Linux, в отличие от Windows, антивирус - это зачастую отдельностоящая
программа, работающая с невысоким уровнем привилегий и делающая что-то одно. Например, проверка
файлов. Так чего мы ждём?

## Немного оговорок

Сразу оговорюсь, что всё изложенное в статье делалось в Qubes 4.1, которая не первой свежести уже.
Следовательно, всё остальное тоже слега устарело. Но общий подход вряд ли изменился к настоящему
моменту.

В этой статье будет много ссылок на официальную документацию, ибо нет смысла пересказывать её в
самой статье. Вопрошающий да ознакомится по ссылке (а страждущий - полностью прочитает документацию
до начала работы с ОС; читать вдумчиво не обязательно, всё равно с первого раза запомнить и понять
сложно, достаточно пробежать по диагонали. Главное, знать где и что искать в случае необходимости.
Но глоссарий - это must read. Остальным придётся поверить на слово. Также здесь будет самый минимум
картинок. Тем более, что не всё можно сделать из пользовательского интерфейса. В каждом примере
командной строки будет упоминаться, в какой виртуальной машине их нужно запускать.

Разберёмся также с терминологией. Нет смысла пересказывать
[официальный глоссарий](https://doc.qubes-os.org/en/latest/user/reference/glossary.html): он
прекрасен в своей точности и полноте. Просто немного подсократим его, а заодно русифицируем. Итак,

* [qube](https://doc.qubes-os.org/en/latest/user/reference/glossary.html#term-qube) - это просто
  куб,
* [standalone](https://doc.qubes-os.org/en/latest/user/reference/glossary.html#term-standalone) -
  отдельностоящий ?одиночка?,
* [template](https://doc.qubes-os.org/en/latest/user/reference/glossary.html#term-template) -
  шаблон,

* [disposable](https://doc.qubes-os.org/en/latest/user/reference/glossary.html#term-disposable) -
  одноразовый,
* [disposable template](https://doc.qubes-os.org/en/latest/user/reference/glossary.html#term-disposable-template) -
  одноразовый шаблон.

Стратегия такова:

1. Создаём одноразовый шаблон для антивируса из шаблона по умолчанию
2. Удаляем всё лишнее из нового шаблона
3. Устанавливаем антивирус в шаблон
4. Немного подстраиваем антивирус, парочку алиасов делаем.
5. На основе шаблона делаем одноразовый куб
6. Проверяем этот самый одноразовый куб
7. Разбираемся с обновлениями базы сигнатур.
8. Заключение

## Начнём, пожалуй

Прежде, чем что-то делать с системой, нужно обновить её до самой последней версии. Если не релизной,
то хотя бы
[пакеты обновить](https://doc.qubes-os.org/en/latest/user/how-to-guides/how-to-update.html).

Выясним, какие шаблоны имеются в нашем распоряжении. Выходим на рабочий стол, нажимаем
Alt+F2, в появившемся окошке Application Finder'а вводим `xfce4-terminal` (автодополнение поможет),
нажимаем Enter. Откроется командная строка самого главного управляющего куба - так называемого
[dom0](https://doc.qubes-os.org/en/latest/user/reference/glossary.html#term-dom0).

Далее даём команду `qvm-ls --fields=name,class | grep TemplateVM`, чтобы получить список шаблонов,
имеющихся в системе, причём интересовать нас будут только шаблоны:

```
[dom0@dom0 ~]$ qvm-ls --fields=name,class | grep TemplateVM
debian-11               TemplateVM
fedora-34               TemplateVM
whonix-gw-16            TemplateVM
whonix-ws-16            TemplateVM
```

В штатной поставке есть приведённые выше шаблоны. Шаблоны *whonix* нам не подходят, они
предназначены для работы с TOR. Возьмём *debian-11* (с таким же успехом можно было взять и
*fedora-34*). Это шаблон из штатной поставки ОС. В нём нет ничего лишнего. И для антивируса это
самое то в качестве основы.

Создаём новый шаблон *antivirus-base* на основе *debian-11*:

```
[dom0@dom0 ~]$ qvm-clone \
> --class=TemplateVM \
> debian-11 \
> antivirus-base
antivirus-base: Cloning private volume
antivirus-base: Cloning root volume
```

Назначим ему метку зелёного цвета:

```
[dom0@dom0 ~]$ qvm-prefs \
> antivirus-base \
> label green
```

Цвета -
[основа графического интерфейса Qubes OS](https://doc.qubes-os.org/en/r4.3/introduction/getting-started.html#color-security).
Есть определённые умолчания, касающиеся выбора меток, и некоторые на этот счёт. Как правило, самый
безопасный куб - чёрного цвета. Самый опасный - красного. Но в целом выбор цвета произволен. Мы
выбрали зелёный, потому что собственно шаблон не несёт угрозы. В то же время этот шаблон так или
иначе будёт иметь выход Интернет, с большой долей вероятности в обход
[штатного механизма обновлений](https://doc.qubes-os.org/en/latest/user/how-to-guides/how-to-update.html)
Qubes OS (об этом позже).

Впрочем, из debian-11 пришло кое-что лишнее. Например, при работе с антивирусом совершенно не нужен
веб-браузер, почтовый клиент и ещё парочка приложений. Запустим эмулятор терминала новоиспеченного
шаблона:

```
[dom0@dom0 ~]$ qvm-run \
> antivirus-base \
> gnome-terminal
```

Откроется привычное окно эмулятора терминала, только с зелёным заголовком.

Удалим ненужные пакеты (обратите внимание, что приглашение изменилось, теперь мы вводим команды не в
dom0, а в только что созданном кубе):

```
user@antivirus-base:~$ sudo apt purge \
> firefox-esr \
> thunderbird \
> evince \
> keepassxc
```

Список ненужного софта можно продолжить. Есть ли в этом выгода? В моём случае, освободилось 500
мегабайт на диске. Также уменьшилась так называемая attack surface. Так что вопрос дискуссионный. Ну
и подчистим немного:

```
user@antivirus-base:~$ sudo apt autoremove
```

Ещё 700 мегабайт освободилось (но тут всё зависит от того, давно ли вы делали этот самый autoremove)

Установим собственно антивирус [ClamAV](https://github.com/Cisco-Talos/clamav) (по крайней мере у 
меня 
https://www.clamav.net/ не открывается, как будто заблокирован) Так что попробуем другой 
какой-нибудь.


-------------------


Просто шаблон нам не подходит, потому что для одноразовых кубов нужен одноразовый шаблон. А
одноразовый шаблон, в свою очередь, это просто куб с вручную "
подкрученным" свойством **template_for_dispvms**. Запутанно, не поспоришь. Но ничего не остаётся как
следующую команду:

```
[dom0@dom0 ~]$ qvm-create \ 
--class=AppVM \
--template=debian-11 \
--label=green \
antivirus-base
```

`--class=AppVM` предписывает создать обычный куб. С выбором шаблона всё понятно.

Проверим, что новый куб действительно был создан:

```
[dom0@dom0 bin]$ qvm-ls --fields=name,class | grep antivirus-base
antivirus-base          AppVM
```

Теперь превратим наш новоиспеченный обычный куб в одноразовый:

```
[dom0@dom0 bin]$ qvm-prefs \
> antivirus-base \
> template_for_dispvms on
```

И ещё раз





```
qvm-features antivirus appmenus-dispvm 1
```

```
qvm-run --dispvm=antivirus --service qubes.StartApp+xterm
```
